# 实验规划

## 核心思路

基于 SD 预训练扩散模型，外挂轻量调制模块，实现**单步多退化图像恢复**。两阶段：

- **Stage 1** — 训练多退化分类器：DINOv2 backbone + Classifier Layer，二者均可训。输出 $\mathrm{logits} \in \mathbb{R}^{B \times C \times 2}$，逐行 softmax 取第一列得 $V \in \mathbb{R}^{B \times C}$
- **Stage 2** — 冻结 SD 主干，仅训练外挂调制模块 + 可选 LoRA。退化特征 $F_{\mathrm{Deg}} = U^T V$，$U \in \mathbb{R}^{C \times d}$ 为可训练字典。固定 timestep 加噪 → 单步去噪 → 输出恢复结果

参考：S3Diff ([2409.17058](https://arxiv.org/abs/2409.17058))、CODSR ([2512.14061](https://arxiv.org/abs/2512.14061))

---

## 实验设计总则

**影响力从大到小逐层贪心调优**：固定下层默认 → 测试上层选项 → 选优胜 → 继承 → 继续。

1. 固定 2,3,4,5 默认 → 测 1a / 1a-v3 / 1b → 选优胜
2. 固定 1（优胜）,3,4,5 默认 → 测 2a~2e → 选优胜
3. 固定 1,2（优胜）,4,5 默认 → 测 3a vs 3b → 选优胜
4. 以此类推

## 1. 调制模块设计

| 选项 | 名称 | registry key | 描述 |
|:---:| --- | --- | --- |
| 1a | ResBlock 内 CrossAttn | `deg_resblock_attn` | F_Deg → TokenGenerator(768→16×64 token) → ResBlock 内 h₂ 作 Q，CrossAttn → γ,β（类似 StableSR 但注入 ResBlock 内部）|
| 1b | CODSR LQFM 调制 | `codsr_lqfm` | LQ → CNN stem → 多尺度特征 → per-layer SFT → α,β（time-aware λ_t）。**注：当前实现为 SimpleSFTModule，无 LQ pixel_unshuffle encoder** |
| 1c | Deg-Aware SFT 调制 | `deg_aware_sft` | **当前主力方案**。LQ CNN 多尺度特征 + F_Deg per-scale channel gate → SFT → γ,β。F_Deg 控制 LQ 特征的通道门控

### 1a vs 1c 对比（当前代码实际实现）

| 维度 | 1a (ResBlockAttnModule) | 1c (DegAwareSFTModule) |
|---|---|---|
| 退化特征 | F_Deg → TokenGenerator(768→16×64) | F_Deg → deg_proj(768→64)→sigmoid gate |
| 注入位置 | ResBlock **内部**（conv1→mod_block→norm2→conv2）| ResBlock **外部**（hook conv2 输出） |
| 注入方式 | CrossAttn(Q=h₂, K/V=tokens) → γ,β | channel gate ⊙ LQ_feat → SFT head → γ,β |
| Hook 层数 | 仅所有 ResBlock（下+中+上）| 所有 conv（conv_in + ResBlock.conv2 + 上下采样 conv），~29层 |
| 关键差异 | 退化 token 参与 attention 计算 | 退化信号仅做通道门控，不参与 attention |
| 参数量 | TokenGenerator + 每 ResBlock 一组 CrossAttn（~20M）| deg_proj×4 + 每 conv 一组 SFT head（~60M） |

**设计判断**：1a 把退化信号直接注入 ResBlock 的 cross-attention，退化 token 可以"查"UNet 的中间特征。1c 的退化信号只做门控——退化信息决定 LQ 特征的哪些通道进入 SFT，但 UNet 本身从不显式感知退化类型。1a 更适合多退化场景（需要区分雾/雨/噪），1c 对单退化更适用（如 StableSR 只做 SR）。倾向 **1a 应为第 1 层的实际优胜者**。

### F_Deg 升级：Scheme B（共通）

两类调制模块均使用：$F_{\mathrm{Deg}} = \mathrm{cls\_token} + \alpha \cdot (V \cdot U)$

- `cls_token`：DINOv2 encoder 直接输出的 [CLS] token（768 维连续表征）
- `V·U`：分类 probs (B,C) @ 可训练 embedding (C,768)——显式退化类型先验
- `α`：可训练标量（初始 10.0），平衡 cls_token（norm~46）和 proto（norm~1）的数值尺度
- DINOv2 encoder+decoder 冻结，惰性构建在 `.to(bf16)` 之后以避免精度损失；`get_deg_feat()` 内部显式 cast 保证无 autocast 的 eval 路径不崩
- **旧版 F_Deg**：仅 $V \cdot U$，3 原型 softmax 加权（有效自由度 2 维）。已弃用。

| 2c  | 200      | |
| 2d  | 250      | |

当前默认：**2a (t=100)**。理由：t 越小越接近 identity 重建，PSNR 偏高；t 越大越需要利用 SD 生成先验。100 是当前平衡点。

## 5. 超参数默认配置

### 5.1 损失函数

| 选项 | 公式 | λ 权重 | 默认 | 说明 |
|:---:| --- | --- |:---:| --- |
| 5.1a | L1 + LPIPS | L1=2.0, LPIPS=5.0 | ✓ | 对齐实验规划默认；LPIPS 占~95% loss梯度，语义偏向感知→PSNR偏低 |
| 5.1b | L1 + LPIPS + GAN | L1=2.0, LPIPS=5.0, GAN=0.5 | | 代码已实现，未验证 |

**已知问题（5.1a）**：LPIPS 5.0 占 loss 梯度 95%，L1 仅占 5%。优化方向严重偏向 VGG 感知空间而非像素忠实，去噪 PSNR 被主动牺牲。调低 LPIPS（0.1~0.2）或降低 L1 权重均可改善 PSNR，但 L1 主导会导致纹理偏软（"平滑"问题）。加 GAN 可缓解平滑但会进一步降 PSNR（GAN 推感知纹理，与 PSNR 冲突）。**见 DOD (AAAI'26) 对比：纯 L1+LPIPS baseline PSNR 23.05→加 CLoRA+DEM→31.87（+8.8dB），证明 loss 权重不是主瓶颈，架构容量才是。**

## 实验设计

> 核心思路：Stage 1（分类器）+ Stage 2（调制模块）→ 在标准 All-in-One 数据集上训练 → 与 SOTA 对比。
> 协议来源：R2R (CVPR 2026)、PromptIR (NeurIPS 2023)、IDR (CVPR 2023) 统一使用同一套数据集标准。
> 训练数据：现阶段使用 FoundIR 子集（10Rain / 08Haze / 14Lowlight），后续扩展到 5D 标准数据。

---

### (1) All-in-One 多退化统一训练

#### 3D All-in-One（对标 R2R Table 1，haze + rain + noise）

> 目标：单一模型同时处理 3 种退化。所有方法在**相同合并数据集**上训练，公平对比。

| 退化类型                 | 训练集                | 数量     | 测试集             | 数量  | 来源                       |
| -------------------- | ------------------ |:------:| --------------- |:---:| ------------------------ |
| Dehazing             | RESIDE (ITS + OTS) | 72,135 | SOTS (outdoor)  | 500 | RESIDE [Li et al. 2018]  |
| Deraining            | Rain100L (train)   | 200    | Rain100L (test) | 100 | [Yang et al. 2017]       |
| Denoising (Gaussian) | BSD400 + WED       | 5,144  | BSD68           | 68  | BSD [Martin et al. 2001] |

> 训练细节（follow R2R/PromptIR）：三数据集合并为一个训练池。无需数据重复（三个数据集数量级相近）。AdamW 优化器。测试用 S3Diff 对齐方式（pad 到 64 reflect，推理后裁回）。指标：PSNR(Y) · SSIM(Y) · LPIPS。

#### 5D All-in-One（对标 R2R Table 2，+ deblur + low-light）

> 目标：在 3D 基础上增加去模糊和低光照增强。

| 退化类型                  | 训练集                | 数量     | 测试集             | 数量    | 来源                   |
| --------------------- | ------------------ |:------:| --------------- |:-----:| -------------------- |
| Dehazing              | RESIDE (ITS + OTS) | 72,135 | SOTS            | 500   | [Li et al. 2018]     |
| Deraining             | Rain100L (train)   | 200    | Rain100L (test) | 100   | [Yang et al. 2017]   |
| Denoising (Gaussian)  | BSD400 + WED       | 5,144  | BSD68 σ=25      | 68    | [Martin et al. 2001] |
| Deblurring (Motion)   | GoPro (train)      | 2,103  | GoPro (test)    | 1,111 | [Nah et al. 2017]    |
| Low-Light Enhancement | LOL-v1 (our485)    | 485    | LOL-v1 (eval15) | 15    | [Wei et al. 2018]    |

> 训练细节：5 个训练集合并。数据重复比 [1, 1, 1, 30, 60]（derain/dehaze/denoise 不重复，deblur 重复 30×，low-light 重复 60× 以大致平衡各任务数据量）。注意数据集命名差异：SOTS = RESIDE 测试集，LOL-v1 = LOL。

---

### (2) Single Task 单任务独立训练（对标 R2R Table 3）

> 目标：分别为每种退化训练独立模型，证明 All-in-One 不会因多任务干扰而严重退化。

| 任务        | 训练集          | 数量     | 测试集               | 数量   | 指标          |
| --------- | ------------ |:------:| ----------------- |:----:| ----------- |
| Dehazing  | RESIDE       | 72,135 | SOTS              | 500  | PSNR / SSIM |
| Deraining | Rain100L     | 200    | Rain100L          | 100  | PSNR / SSIM |
| Denoising | BSD400 + WED | 5,144  | CBSD68 σ=15,25,50 | 68×3 | PSNR / SSIM |

> 对比对象：AirNet、PromptIR 的单任务版本。这是检验"多任务干扰"的基准线——如果 All-in-One 和 Single-Task 差距太大，说明任务冲突严重。

---

### (3) OOD Generalization 跨域泛化（对标 R2R Fig.3 任务扩展分析）

> 目标：不做额外 fine-tune，直接用训练好的 3D/5D 模型测**未见过**的退化类型或真实世界数据。

| 退化类型               | 测试集                      | 数量      | 说明                  |
| ------------------ | ------------------------ |:-------:| ------------------- |
| Motion Deblurring  | RealBlur-R               | ~1,000  | 真实模糊，0 训练           |
| Defocus Deblurring | CUHK, PixelDP            | ~700    | 散焦模糊                |
| Deraining (real)   | RealRain-1K              | ~1,000  | 真实雨                 |
| Desnowing          | Snow100K-real            | ~10,000 | 真实雪                 |
| Dehazing (real)    | RTTS                     | ~4,300  | 真实雾                 |
| Low-Light (real)   | DICM, LIME, MEF, NPE, VV | ~300    | 真实低光（无 GT，用 NR-IQA） |

> 这是你区别于 CNN/Transformer 方法的核心优势——SD 2.1 的生成先验能在没见过的退化上泛化，而 PromptIR/AirNet 等做不到。
